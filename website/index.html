Frequently Asked Questions

Common questions from FastAPI developers evaluating Catzilla.

Is Catzilla really compatible with FastAPI syntax?

Yes. Catzilla maintains 95% syntax compatibility with FastAPI for routing, validation, and response handling. The same decorators (@app.get, @app.post), validation models (BaseModel, Field, Path, Query), and StreamingResponse work identically. Dependency injection uses Depends("service_name") with string-based service names and a @service decorator. Middleware uses @app.middleware(priority=..., pre_route=True) with priority ordering. Key differences: (1) handlers receive request as the first parameter, (2) the import changes from fastapi to catzilla, and (3) no separate ASGI server is needed.

How much faster is Catzilla than FastAPI?

In the latest reproducible wrk benchmark artifacts in this repository on Apple Silicon (macOS arm64), Catzilla reaches 82,817 requests/second for a single-worker hello-world endpoint versus FastAPI's 35,154 req/s, and 190,612 req/s with 10 workers versus FastAPI's 109,212 req/s. For query-parameter endpoints in the same benchmark set, Catzilla is up to 4.3× faster. The benchmark artifacts are open source and reproducible.

Why is Catzilla faster than FastAPI?

Catzilla's HTTP engine is written in C using libuv (the same event loop behind Node.js) and llhttp (the same HTTP parser used by Node.js). Routing uses an O(log n) trie in C. Memory management uses jemalloc with specialized arenas. Validation runs through a C-accelerated engine instead of Pydantic's Python runtime. This eliminates Python overhead from the entire hot path while keeping the developer-facing API in pure Python.

Does Catzilla require Pydantic or Uvicorn?

No. Catzilla ships its own validation engine, HTTP server, and routing stack, so you do not need Pydantic, Starlette, or Uvicorn to use it. It supports Pydantic-style BaseModel syntax through its built-in validator. The package does currently declare a small runtime dependency on psutil, so the accurate claim is a minimal dependency footprint, not literally zero dependencies.

What Python versions and platforms are supported?

Catzilla supports Python 3.9 through 3.14 with pre-built binary wheels for Linux (x86_64, manylinux2014), macOS (Intel x86_64 and Apple Silicon arm64), and Windows (x86_64). ARM64 Linux is supported via source builds. The framework is tested across all platforms in CI.
